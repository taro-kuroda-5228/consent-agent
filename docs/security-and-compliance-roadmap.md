# Security and compliance roadmap

- Positioning: physician-support workflow, not autonomous consent, diagnosis, indication, or signed consent capture.
- PHI policy:
  - Demo: anonymous only; upload blocker rejects PHI-like content with category/risk/next choices.
  - PoC: DPA/BAA-equivalent terms, region controls, encryption, immutable audit, backup/retention policy.
  - Hospital: self-host/on-prem option with institutional governance.
- Audit: model mode, explanation version, selected evidence IDs, understanding evaluations, physician review status, export events, and export hash in future iterations.
- Regulatory: evaluate SaMD applicability; extend development record toward IEC 62304-style traceability if clinical production use advances.
